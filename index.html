REFLECTION

In this lab, I learned how to set up and use GitHub as a version control and hosting platform for a simple website.
I created a new repository, configured it correctly with a README and .gitignore, and then built a basic HTML
webpage saved as index.html. I used Git commands such as git add, git status, and git commit to track my files
locally before pushing them to GitHub with git push. I also learned how to manage branches and ensure my local
repository matched GitHub’s structure. After uploading my files, I enabled GitHub Pages to deploy the website publicly,
allowing it to be accessed from anywhere via a web link. Finally, I improved the webpage with basic HTML and CSS
changes, added images and text from my working document, and documented each step with screenshots.
This lab demonstrated how GitHub is not just a storage platform but a powerful tool for managing projects, tracking changes,
and deploying work efficiently. In industry, these skills are essential for software development teams where
multiple people work on the same project. Version control prevents loss of work, keeps a history of changes, and
allows collaboration without conflict. The ability to deploy a website through GitHub Pages also shows how developers
can quickly publish updates and share results with clients or team members. Overall, this process reflects real-world
workflows used in software engineering, DevOps, and web development, where code is developed locally,
managed through Git, and deployed through cloud-based services.
